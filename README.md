# 動画仕分けボード

YouTube動画を「見る前に仕分ける」ための、GitHub Pages対応の軽量静的Webアプリです。特定の発信者専用ではなく、気になるチャンネルや動画を手動で登録し、要約・重要ポイント・視聴判断・自分向け示唆をカード形式で確認できます。整理テンプレートによって、発信用・調査用・企画用に使い分けられます。

## アプリ概要

- HTML / CSS / JavaScriptのみで動作します。
- サンプル動画データは `videos.json` で管理します。
- v2では画面上の入力フォームから動画を追加し、ブラウザの `localStorage` に保存できます。
- v2.1では、まずURL・タイトル・メモだけで登録し、あとから要約や視聴判断を追記できます。
- YouTubeの文字起こし・本文を保存し、AI整理プロンプトに含められます。
- 整理テンプレートを選ぶことで、AI整理プロンプトを調査用・SNS投稿用・YouTube企画用・note記事用などに切り替えられます。
- YouTube URLを貼ると、動画IDを抽出してサムネイルを自動表示します。可能な場合だけ、タイトルとチャンネル名も補助的に自動取得します。
- スマホでも見やすいカード型UIです。
- ダークネイビー基調のデザインです。
- 以下の条件で絞り込みできます。
  - 視聴判断別
  - 深掘り候補あり
  - 投稿化候補あり


## AI整理プロンプトをコピーする使い方

保存済み・サンプルを含む各動画カードには **AI整理プロンプトをコピー** ボタンがあります。ボタンを押すと、そのカードの動画タイトル、チャンネル名、動画URL、気になった理由・メモ、選択テンプレート名を含むAI用プロンプトがクリップボードにコピーされます。コピーが完了すると、ボタン表示が一時的に **コピーしました** に変わります。

コピーしたプロンプトは ChatGPT / Gemini / Claude などに貼り付けて、以下を整理するために使えます。

- 動画本文・文字起こし・字幕を確認できたかを前提にした判定
- 動画を全部見る前の視聴判断
- 重要ポイントやテーマとの関係
- X、Threads、note、YouTubeショート、通常動画など発信に使える角度
- 追加リサーチすべき論点、公式情報、統計、反対意見
- この動画だけを根拠に発信すると危ない点
- ボードに戻しやすい要約、重要ポイント、自分向け示唆、整理状態

文字起こし・本文を保存しているカードでは、プロンプト内に全文と「以下の文字起こしをもとに整理してください」という指示が含まれます。文字起こし・本文が未入力のカードでは、「本文なし。タイトル・概要情報からの仮判定」と明記され、内容を見た前提で要約しないよう注意文が入ります。

### YouTube文字起こしを使った整理手順

1. YouTubeの動画ページで文字起こしを表示し、必要な範囲をコピーします。
2. yt-boardの追加フォーム、または追加データカードの編集フォームにある **文字起こし・本文** 欄へ貼り付けます。
3. 必要に応じて **取得方法メモ** に「YouTube文字起こしをコピー」「概要欄を転記」など、本文の出どころをメモします。
4. 動画カードの **AI整理プロンプトをコピー** を押します。
5. コピーしたプロンプトをAIに投げて、要約・視聴判断・深掘り判断を行います。

スマホでも押しやすいよう、ボタンはカード幅に合わせて大きく表示されます。

## 整理テンプレート

カードには選択中の整理テンプレートが小さなラベルで表示されます。**AI整理プロンプトをコピー** を押すと、選んだテンプレートに合わせた観点を含むプロンプトをクリップボードへコピーできます。

初期テンプレートは以下です。

- 汎用リサーチ: 要点、論点、見る価値、追加調査
- SNS投稿ネタ化: X/Threads向けの切り口、炎上注意点、投稿案
- YouTube企画化: 動画企画、ショート化、台本の切り口
- note記事化: 記事テーマ、見出し、読者に刺さる論点
- AIニュース分析: 事実確認、一次情報、リスク、活用可能性
- 管理職・現場視点: 現場で起きそうなズレ、管理職への示唆
- 40代のリアル: 管理職、現場、AI活用、働き方、SNS、世代間ギャップの観点

## v2.1: まず軽く登録して、あとから整理する使い方

動画を見つけた瞬間に登録しやすいよう、v2.1では追加フォームを **最低限入力** と **詳細入力** に分けました。

### 最低限入力で登録する

1. トップ画面の **動画を追加** ボタンを押します。
2. 通常表示される以下の項目を入力します（文字起こし関連は任意です）。
   - 動画URL
   - チャンネル名
   - 動画タイトル
   - 気になった理由・メモ
   - 文字起こし・本文（任意）
   - 取得方法メモ（任意）
3. URL欄からフォーカスを外すか **動画IDを取得** を押すと、対応URLから動画IDを抽出し、サムネイルを表示します。GitHub Pagesではタイトル・チャンネル名の自動取得は失敗する場合があるため、まずはサムネイル自動表示＋手入力で運用します。
4. **あとで整理する** を押すと、詳細項目が未入力でも未整理の動画として `localStorage` に保存され、あとから編集・整理する前提で登録できます。

この登録方法では、要約・重要ポイント・自分向け示唆は `未整理`、視聴判断は `D 保留`、深掘り候補は `未判定`、投稿化候補は `なし` として保存されます。公開日は未入力のまま保存できます。整理テンプレートを未指定にした既存データは `汎用リサーチ` として扱われます。

### GitHub PagesでのYouTube情報取得について

GitHub Pagesでは静的サイトからYouTube oEmbedや外部APIへ直接fetchすると、CORSなどの制約でタイトル・チャンネル名の自動取得が失敗する場合があります。そのため、このアプリではブラウザだけで確実にできる動画ID抽出とサムネイル自動表示を優先し、タイトル・チャンネル名は手入力できる運用にしています。oEmbed取得は補助機能であり、失敗しても動画登録は止まりません。

### 詳細入力まで済ませて登録する

追加時点で整理できる場合は、フォーム内の **詳細を入力する** を押すと以下を入力できます。

- 公開日
- 要約
- 重要ポイント
- 自分向け示唆
- 整理テンプレート
- 視聴判断（A 全部見る / B 該当箇所だけ見る / C 要約で十分 / D 保留）
- 深掘り候補（あり / なし）
- 投稿化候補（X / note / YouTube / なし）

登録後のカードでは、チャンネル名・動画タイトル・動画URL・気になった理由・メモ・状態（未整理 / 整理済み）を先に確認できます。既存の詳細項目があるデータは、従来どおりカード内に表示されます。

## v2: 画面から動画を追加する方法

GitHub Pagesは静的サイトのため、画面から直接 `videos.json` を書き換えることはできません。v2では、追加した動画をブラウザの `localStorage` に保存し、`videos.json` のサンプルデータと合わせて表示します。

### 動画の追加

1. トップ画面の **動画を追加** ボタンを押します。
2. 通常フォームでは、動画URL・チャンネル名・動画タイトル・気になった理由やメモだけを入力します。
3. 用途に合わせて **整理テンプレート** を選びます。未指定の既存データは `汎用リサーチ` として扱われます。
4. 要約や視聴判断まで入れたい場合だけ、**詳細を入力する** を押して詳細項目を開きます。
5. **あとで整理する** を押すと、詳細項目が未入力でも追加データとしてカード一覧に表示されます。未整理データはあとから編集・整理する前提で登録できます。

### 追加データの削除

追加した動画カードには **削除** ボタンが表示されます。削除すると、そのブラウザの `localStorage` から該当データが消えます。`videos.json` のサンプルデータは削除できません。

### JSONエクスポート / インポート

- **追加データをエクスポート** を押すと、`localStorage` に保存されている追加データだけをJSONファイルとしてダウンロードできます。
- **JSONをインポート** から、エクスポートしたJSONファイルを読み込めます。インポートすると、現在の追加データは読み込んだ内容で置き換わります。

### 注意点

- `localStorage` はブラウザ単位の保存です。別端末・別ブラウザには自動同期されません。
- ブラウザのサイトデータを削除すると、追加した動画も消える可能性があります。必要に応じてJSONエクスポートでバックアップしてください。
- 将来的にGitHub APIや外部DBへ拡張しやすいよう、サンプルデータ（`videos.json`）と追加データ（`localStorage`）を分けて扱っています。

## `videos.json` の編集方法

`videos.json` は動画オブジェクトの配列です。新しい動画を追加する場合は、既存のオブジェクトをコピーして末尾に追加し、各項目を書き換えてください。

```json
{
  "id": "sample-004",
  "channel": "チャンネル名",
  "title": "動画タイトル",
  "url": "https://www.youtube.com/watch?v=...",
  "publishedDate": "2026-05-12",
  "summary": "動画の要約",
  "points": [
    "重要ポイント1",
    "重要ポイント2",
    "重要ポイント3"
  ],
  "insight": "自分向け示唆",
  "transcript": "YouTubeの文字起こしや本文メモ",
  "transcriptSourceNote": "YouTube文字起こしをコピー",
  "organizeTemplate": "general",
  "watchDecision": "A",
  "deepDive": "あり",
  "contentUse": "note"
}
```

### 入力ルール

- `id` / `title` / `channel` / `url` だけを必須項目として扱います。古いデータや未整理データで詳細項目がない場合も、読み込み時に初期値を補完します。
- `summary` の未入力時は `未整理`、`points` の未入力時は `["未整理"]`、`insight` の未入力時は `未整理` として扱います。
- `watchDecision` は以下のいずれかを入力します。
  - `A`: 全部見る
  - `B`: 該当箇所だけ見る
  - `C`: 要約で十分
  - `D`: 保留
- `deepDive` は `あり` / `なし` / `未判定` のいずれかを入力します。未入力や古い `true` / `false` / `null` 形式のデータは読み込み時に補完・変換します。
- `contentUse` は `X` / `note` / `YouTube` / `なし` のいずれかを入力します。未入力時は `なし` として扱います。
- `organizeTemplate` は `general` / `sns` / `youtube` / `note` / `aiNews` / `manager` / `fortiesReal` のいずれかを入力します。未入力や古いデータの場合は `general`（汎用リサーチ）として扱います。
- `transcript` は文字起こし・本文です。未入力時や既存データに項目がない場合は、読み込み時に空文字として補完します。
- `transcriptSourceNote` は取得方法メモです。未入力時や既存データに項目がない場合は、読み込み時に空文字として補完します。
- 追加フォームから保存したデータには、対応YouTube URLから抽出した `videoId` と `thumbnailUrl` も保存されます。古いデータのようにこれらがない場合も、URLからサムネイルを補って表示します。
- JSONの最後の項目にはカンマを付けないようにしてください。

## GitHub Pagesでの公開方法

1. このリポジトリをGitHubにプッシュします。
2. GitHubのリポジトリ画面で **Settings** を開きます。
3. **Pages** を選択します。
4. **Build and deployment** の **Source** で `Deploy from a branch` を選びます。
5. **Branch** で公開したいブランチと `/ (root)` を選び、保存します。
6. 数分後に表示されるGitHub PagesのURLへアクセスします。

`index.html`、`style.css`、`app.js`、`videos.json` がリポジトリ直下にあるため、追加ビルドなしでそのまま公開できます。
